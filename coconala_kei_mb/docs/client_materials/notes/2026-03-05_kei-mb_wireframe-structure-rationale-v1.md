# Kei mb様 ワイヤーフレーム初稿の構造根拠（v1）

作成日: 2026-03-05  
対象: LP初稿（最大8セクション）

## 1. この構造にした理由（結論）
- 要件定義の固定条件「1ページ / 最大8セクション / 一次CV=フォーム送信 / スマホ最適化」を守りつつ、参考LPの強みである「FV即入力（A）」「少項目開始（F）」を優先するため。
- 参考LPの長尺構成をそのまま複製せず、CVに効く要素だけを8セクションへ圧縮したため。

## 2. 採用する8セクション（役割つき）
1. **FV + トップフォーム（3項目）**
- 役割: スクロール前に入力開始。
- 根拠: A/F重視、入力離脱の抑制。

2. **悩み共感（電話ラッシュ・比較の手間）**
- 役割: 「自分ごと化」を作る。
- 根拠: 主要訴求「電話ラッシュなし」「比較負荷軽減」。

3. **解決策（3ステップ）**
- 役割: 仕組み理解と不安低減。
- 根拠: 送信後導線の明確化（要件6章）。

4. **価格差イメージ（例示 + 注記）**
- 役割: 比較する価値を視覚化。
- 根拠: 断定回避の法務要件（NG表現回避）。

5. **選ばれる理由 Top3**
- 役割: 価格以外の判断材料を提示。
- 根拠: USP Top3（地域密着/高品質リード/料金設計）。

6. **対応エリア・対応タイプ（表）**
- 役割: 対象可否の即時判断。
- 根拠: 近畿圏訴求、ターゲット6区分。

7. **FAQ（5問）**
- 役割: 送信直前の阻害要因を除去。
- 根拠: 要件定義11章のFAQたたき台。

8. **最終CTA + 法務リンク + 運営情報**
- 役割: 最終行動喚起と信頼担保。
- 根拠: CTA統一、規約/プライバシー必須リンク要件。

## 3. 参考LPからの採用/非採用
### 3.1 採用した考え方
- FVでの即入力開始
- 初期入力項目を少なくする
- 比較価値の可視化（価格差・事例・理由）
- FAQから最終CTAへの一本道導線

### 3.2 今回あえて外した要素（理由）
- ランキング大型ブロック、速報ティッカー、訪問モーダル、過多なCTA反復
- 理由: 「最大8セクション」「修正2往復」「10万円スコープ」内で、実装と運用の再現性を優先するため。
